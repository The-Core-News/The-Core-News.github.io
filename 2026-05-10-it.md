# Bun, Rust 재작성 99.8% 호환 달성 — Linux 한정, 전면 전환보다 파일럿 도입 시점 재검토할 때  **2026년 5월 10일**  Bun의 창시자 Jarred Sumner가 오늘(5월 10일) 자신의 X(구 Twitter) 계정을 통해 Bun의 Rust 재작성 실험 브랜치가 Linux x64 glibc 환경에서 99.8%의 테스트 호환성을 달성했다고 밝혔다. 이 수치는 Sumner 본인의 직접 발표로, 현재까지 제3자 독립 검증은 이루어지지 않았다.  결론부터 말한다: 지금 당장 Node.js 프로덕션을 Bun으로 전환하라는 신호가 아니다. 그러나 Bun 마이그레이션 타이밍을 진지하게 검토하지 않던 팀이라면, 이번 이정표를 파일럿 계획의 출발점으로 삼을 수 있다.  ---  ## 99.8%의 맥락: 무엇을 말하고, 무엇을 말하지 않는가  Sumner가 발표한 99.8% 호환성 수치는 **Linux x64 glibc 환경으로 한정**된다. 국내 개발팀의 실제 환경—Windows WSL2, macOS Apple Silicon(M시리즈), AWS Graviton(ARM)—에서 동일한 수치를 기대할 근거는 현재 없다. 크로스플랫폼 호환성 데이터는 아직 공개되지 않았다.  해당 브랜치는 공식 릴리스가 아닌 **실험적(experimental) 브랜치**다. Sumner 본인도 이를 명시하고 있으며, 프로덕션 준비 완료 타임라인은 Bun 측에서 공식 발표한 바 없다.  ---  ## Rust 재작성이 의미하는 것  성능 수치보다 중요한 변화가 있다. Bun의 코어는 지금까지 C++로 작성되어 있었다. Rust로의 전면 재작성은 두 가지를 뜻한다.  첫째, **메모리 안전성**이다. C++의 고질적인 메모리 오류 클래스—use-after-free, buffer overflow—를 언어 수준에서 차단한다. 런타임의 장기 안정성에 직접 영향을 준다.  둘째, **장기 유지보수성**이다. 오픈소스 프로젝트에서 Rust 생태계의 기여자 저변은 C++보다 빠르게 성장하고 있다. Bun이 이 전략을 택한다는 것은, 단기 성능 경쟁이 아니라 2~3년 후 생태계 지속 가능성에 베팅한다는 의미다.  Rust 재작성 버전의 구체적인 성능 벤치마크는 아직 공개되지 않았다. 현재로서는 이정표 달성 자체가 뉴스이며, 성능 개선 폭을 논하는 것은 시기상조다.  ---  ## 국내 Node.js 팀을 위한 실용적 판단  국내 스타트업과 핀테크 팀 사이에서 Bun에 대한 관심은 높지만, 실제 프로덕션 전환율은 여전히 낮다. 주된 이유는 두 가지다: Windows 개발 환경 지원의 불안정성, 그리고 NestJS·Prisma 기반 기존 코드베이스와의 호환성 불확실성. 이 두 가지는 이번 발표로도 해소되지 않았다. NestJS, Prisma, Fastify 등 국내 주류 스택과의 호환성은 별도 검증이 필요하다.  현실적으로 권장할 수 있는 경로는 **점진적 도입**이다. 전체 서버 런타임을 교체하는 것이 아니라, 다음 워크로드부터 시작하는 방식이다:  - **번들러 교체**: 기존 webpack, Vite 대신 `bun build`를 먼저 평가한다. - **테스트 러너 교체**: Jest 대신 `bun test`를 CI 파이프라인에 시험 도입한다. - **패키지 매니저 교체**: `bun install`은 현재 가장 성숙한 Bun 기능이며, 호환성 리스크가 가장 낮다.  이 세 가지는 프로덕션 런타임 리스크를 지지 않으면서도 Bun의 실질적 이점을 먼저 경험할 수 있는 경로다.  ---  ## 지금 당장 할 일  Bun Rust 재작성 브랜치의 진전 상황을 추적하려면 [Bun 공식 GitHub 릴리스 페이지](https://github.com/oven-sh/bun/releases)를 구독해 두는 것이 현실적이다. Sumner의 개인 계정 발표는 이정표 알림에는 유용하지만, 프로덕션 도입 결정의 근거로 삼기에는 독립 검증이 부족하다.  이번 소식을 '마이그레이션 시작 신호'로 해석하는 것은 앞서 나간 판단이다. 그러나 Bun이 장기 생존 가능한 프로젝트인지 여부에 회의적이었던 팀이라면, 이번 이정표는 그 질문에 대한 긍정적 답변이다. 파일럿 계획을 구체화해도 좋을 시점이다.  ---  *출처: Jarred Sumner(@jarredsumner) X 게시물, 2026-05-10 / 내부 에디터 브리프(candidates.json, selected.json).
